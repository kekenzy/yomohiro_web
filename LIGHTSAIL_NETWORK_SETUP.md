## 🔒 HTTPS（ポート443）を有効にする手順

HTTPSでアクセスするには、AWS Lightsailのネットワーク設定でポート443を開く必要があります。

### 1. AWS Lightsailコンソールでポート443を開く

1. [AWS Lightsail Console](https://lightsail.aws.amazon.com/)にアクセス
2. インスタンス（`yomohiro-web`など）を選択
3. 「ネットワーキング」タブをクリック
4. 「ファイアウォール」セクションで「ルールを追加」をクリック
5. 以下の設定でルールを追加：
   - **アプリケーション**: カスタム
   - **プロトコル**: TCP
   - **ポート範囲**: 443
   - **ソース**: 任意の場所（0.0.0.0/0）
   - **説明**: HTTPS
6. 「保存」をクリック

### 2. 既存のルールを確認

以下のポートが開いていることを確認：
- **ポート22**: SSH（既に開いているはず）
- **ポート80**: HTTP（既に開いているはず）
- **ポート443**: HTTPS（追加が必要）

### 3. 設定確認

ポート443が開いたら、以下で確認：

```bash
# ローカルからテスト
curl -k -I https://54.178.68.240/

# ブラウザでアクセス
# https://18.183.91.131
```

### 4. 注意事項

- 自己署名証明書のため、ブラウザで警告が表示されます
- 「詳細設定」→「続行」でアクセスできます
- 本番環境では、Let's Encryptの証明書を使用することを推奨します

## 📋 現在の設定状態

- ✅ Nginx: ポート443でリッスン中
- ✅ SSL証明書: 自己署名証明書を生成済み
- ✅ Django設定: HTTPSを有効化済み
- ⚠️ Lightsailネットワーク: ポート443を開く必要があります

## 🔍 トラブルシュート: 静的 IP では開くがドメインでは開かない

**原因の多くは DNS が Lightsail の静的 IP を向いていないことです。**  
ブラウザはドメイン名でアクセスするため、**A レコード（および必要なら AAAA）**が本番サーバのグローバル IP と一致している必要があります。

### 確認コマンド（ローカルまたはサーバで実行）

```bash
dig +short yomohirokan.com A
dig +short www.yomohirokan.com A
```

**Lightsail の静的 IP（例: `54.178.68.240`）と同じになること**を確認してください。別の IP が返る場合は、レジストラまたは Cloudflare の DNS で **A レコードを静的 IP に修正**してください（`www` 用の CNAME / A も同様）。

### Cloudflare を使う場合

- DNS の **A レコードのコンテンツ**を **Lightsail の静的 IP** にする（プロキシ ON/OFF は運用に合わせてよい）。
- SSL/TLS のモードが「フル」または「フル（厳密）」のとき、オリジンは Nginx の証明書設定（リポジトリの `config/nginx.conf` や `SSL_SETUP.md`）と整合させる。

### サーバ側が正常かの切り分け

次が 200 ならアプリと Nginx は動いています（ドメインではなく IP 直打ち）。

```bash
curl -sI http://54.178.68.240/
```

ここが 200 なのにドメインだけ失敗する場合は、**まず DNS の向き先**を疑ってください。
